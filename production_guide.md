# 工厂人员异常行为监控系统 — 生产级扩展与精度优化指南

本指南旨在指导开发与工程团队如何将当前基于规则和仿真测试的 MVP 雏形系统，扩展为支持多路并发、高可用、低延迟的**生产级工业应用**，并详细阐述如何从算法和工程层面**最大化提升检测精度与系统降噪**。

---

## 一、 如何将系统扩展为生产级应用 (Production-Scale Architecture)

当前的系统采用了“单线程拉流 + 同步推理”的架构，适用于单路摄像头验证，但在生产环境下（数十路甚至上百路摄像头、7x24小时稳定运行、对接企业业务系统）必须进行架构重构。

```mermaid

    subgraph 视频拉流与分发层

    end

    end
    
    subgraph 业务研判与告警闭环

        H -->|数据持久化| I[(PostgreSQL 历史库)]
```

### 1. 多路视频流并发管理与分布式解码
* **流媒体中转代理**：在摄像头与算法之间部署高性能开源流媒体服务器（如 **SRS** 或 **ZLMediaKit**）。通过 RTMP/RTSP 统一拉流并分发，解决直接拉取摄像头原流导致的连接数上限和画面花屏问题。
* **进程级并发/微服务化**：避免使用 Python 单进程管理多路流。应采用**微服务架构**，将“拉流解码”、“模型推理”、“业务判定”解耦。每一路监控视频由一个独立的容器化轻量拉流进程处理，解码后的帧数据通过共享内存（IPC）或高效队列（Redis Stream）传送给推理池。

### 2. 推理流水线极致优化 (Triton + TensorRT)
* **TensorRT 部署加速**：将 PaddlePaddle 推理模型转换为 ONNX 格式，进而编译为 **NVIDIA TensorRT Engine**，使用 **FP16** 甚至 **INT8** 量化运行。这可使 PP-YOLOE 和 PP-TinyPose 的推理速度提升 3~5 倍。
* **按需惰性推理 (Lazy Inference)**：
  * **姿态估计按需触发**：仅在目标人体的位移速度极低（疑似静止）或检测框宽高比异常（疑似摔倒）时，才对此目标运行 PP-TinyPose 骨骼点推理。处于正常行走的行人不运行姿态估计。
  * **OCR 识别按需触发**：OCR 只在目标脚底进入“抓拍门 (Capture Zone)”且未绑定身份时运行。离开抓拍区后，OCR 模块完全休眠，极大释放显存与算力。
* **推理服务集群**：使用 **Triton Inference Server** 统一托管所有深度学习模型，支持多 GPU 自动负载均衡、动态批处理 (Dynamic Batching)，大幅提升吞吐量。

### 3. 生产级告警管理与数据闭环
* **告警抑制与降噪机制**：在业务逻辑层引入**滑动时间窗口防抖**。对于同一个工人的同一种告警（例如低头玩手机），在触发后进入冷却期（如 5 分钟），在此期间不重复发送通知，避免微信/钉钉消息轰炸。
* **短视频录制与回溯**：系统检测到告警后，除了抓拍带骨骼绘制的现场图外，应利用环形缓冲区保存**告警前 10 秒 + 告警后 10 秒的视频切片**，将其上传至云存储（如 MinIO），并在告警详情中附带播放链接，以便管理人员回溯判定。
* **进程守护与自我修复**：采用 **Supervisor**、**PM2** 或 Kubernetes 的 Liveness 探针，对拉流与推理进程进行 24 小时监控，一旦检测到摄像头断网超时或显存溢出崩溃，执行秒级自动重启。

## 二、 核心算法检测精度优化手段 (Accuracy Optimization)

在实际工厂中，光照变化、摄像头死角、起皱的衣服、行人的复杂交错会产生漏检和误报。以下是工业级精度的优化方案：

### 1. 行人检测与多目标跟踪 (MOT) 优化
* **垂直场景模型微调 (Fine-tuning)**：PP-YOLOE 官方权重基于开源数据集，对工厂俯视视角、安全帽配戴、高反光背心的行人识别率不佳。建议在现场录制 5000 帧左右图像，人工标注后对行人检测器进行微调，能极大降低漏检率。
* **结合行人 ReID 特征**：单纯依靠 ByteTrack 的 IoU 空间匹配，在两人身体重叠 80% 以上时极易发生 ID Switch。应当引入轻量级 **ReID 特征提取网络**（如在 ByteTrack 框架中融入 MobileNet-ReID 特征），在 IoU 匹配失败时计算人体外观特征余弦相似度，锁死目标身份。

### 2. 行为分析算法的鲁棒性优化
#### A. 长时间低头（玩手机）判定优化
* **三维相机标定**：单纯在 2D 像素面上计算“鼻子-颈部”夹角，会随员工面朝方向（正面、侧面、背面）及摄像头安装倾角的变化而产生巨大误差。
  * **方案**：使用 **PnP 算法** 结合人脸/五官关键点，将 2D 骨骼点还原为 3D 头部姿态角（Pitch 俯仰角，Yaw 偏航角）。
* **时序骨架动作识别 (ST-GCN)**：用手工硬编码规则难以区分“低头走路看路”与“站立玩手机”。可在第一阶段后，采集“玩手机”、“看路”、“捡东西”的骨架序列，训练一个轻量级 **时序图卷积网络 (ST-GCN)** 进行端到端分类，准确率可从 80% 提升至 95% 以上。

#### B. 摔倒检测判定优化
* **时序突变检测**：单纯靠单帧的宽高比和高度差极易将“蹲下工作”或“弯腰系鞋带”误判为摔倒。
  * **方案**：引入时序滑动窗口（如 1.5 秒内，约 38 帧）。计算人体框高度降低的**加速度**，以及质心下坠速度。只有“高度瞬间骤降 + 维持横向水平姿态”才触发摔倒告警。

### 3. 号码绑定与 OCR 精度优化
在工业现场，衣服起皱、摄像头反光导致 PaddleOCR 难以直接识别出细小的胸前号码。

* **两阶段识别（检测 + 识别）**：
  * **文字区域精准剪裁**：不要直接将上半身丢给 OCR。应先训练一个极轻量的 **YOLOv8-pose/obbox 目标检测器**，专门用来定位衣服上的**数字号码框**和**安全帽上的号码框**。
  * **号码透视变换校正**：通过号码框的倾斜角度，对其执行透视变换（WarpPerspective）拉直，再送入 OCR 识别网络。
* **OCR 模型专用字形微调**：
  * 用工厂统一定制的反光背心上的字形（如标准的 Arial Black 字体数字）自动生成数十万张包含各种褶皱、阴影、遮挡的**虚拟合成数字图像**。
  * 使用这些合成数据微调 PaddleOCR 的识别器（CRNN/SVTR 网络），能让特定字体数字的识别率直接达到 **99.2%**。
* **工业级投票与纠错词典**：
  * **档案关联纠错**：建立在岗员工“白名单号码池”（如 `['01', '03', '08', '12', ...]`）。若 OCR 偶尔误识为 `"0B"`，通过编辑距离算法计算最邻近词，自动纠正为 `"08"`。
  * **高频多数投票**：将抓拍门内的有效投票阈值从“8帧中占75%”优化为结合“OCR 置信度权重”的加权投票法，置信度越高的帧，票权比重越大。

### 4. 遮挡状态机与抗干扰微调
* **动态锚点有效半径**：离摄像头近的目标，其像素位移范围大，离摄像头远的目标，像素位移小。因此，静态空间锚点（Static Spatial Anchor）的匹配半径应根据**逆透视矩阵（IPM）计算出的实际物理距离（米）**来恒定设定（如半径统一设为 0.8 米），而非像素半径，从而完美适应近景与远景。
* **时间段防漂移**：在状态机中增加对“遮挡物速度”的监控。如果遮挡物（如路过行人）的速度极快，表明这是一次短暂的掠过式遮挡，计时挂起容差时间可缩短；若是两个静止目标相交（如两人站立聊天），则触发聚集判定，挂起原有的单人静止计时，启动聚集计时。
