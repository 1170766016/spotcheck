# 工厂人员行为监控系统 — 2人AI应用开发团队的真实难点评估

> **团队画像**：2人，AI应用开发方向，主要靠大模型辅助生成代码，非底层算法研究背景
> **评估日期**：2026年6月
> **核心结论**：部分功能能做，但有几个坎必须正视

---

## 一、先说结论

| 问题 | 答案 |
|------|------|
| 我们能不能做？ | **部分能做，部分做不了** |
| 最大的障碍是什么？ | **不是写代码，是调试和部署** |
| 大模型能帮多少？ | **写业务逻辑帮很多，调AI模型帮不上** |
| 2个人够吗？ | **做 Demo 够，做生产系统不够** |

## 二、大模型能帮你做什么？不能帮你做什么？

### 2.1 大模型能帮的（约占 30% 工作量）

| 工作 | 大模型能帮的程度 | 说明 |
|------|----------------|------|
| FastAPI 后台接口 | ⭐⭐⭐⭐⭐ | CRUD、数据库、鉴权，闭着眼写 |
| 前端管理看板 | ⭐⭐⭐⭐⭐ | 告警列表、统计图表，大模型很擅长 |
| 告警推送（企微/钉钉） | ⭐⭐⭐⭐⭐ | 调 API 发通知，大模型直接生成 |
| 视频流拉取（RTSP） | ⭐⭐⭐⭐ | OpenCV 读 RTSP 流，代码量小，大模型能写 |
| 数据库设计 | ⭐⭐⭐⭐ | 表结构、查询语句，大模型擅长 |
| 部署脚本（Docker等） | ⭐⭐⭐⭐ | 基础部署，大模型能写 |

### 2.2 大模型帮不了的（约占 70% 工作量）

| 工作 | 大模型帮不上原因 | 你需要面对的 |
|------|----------------|------------|
| **模型跑不起来** | PaddleDetection/YOLO 的环境安装、依赖冲突、版本兼容 | **这是第一个坎，可能卡你 1~2 周** |
| **模型效果不好** | 检测不准、漏检、误检，需要调参数、换模型、调阈值 | **大模型看不到你的摄像头画面，帮不了** |
| **跟踪ID跳变** | ByteTrack 参数敏感，不同场景差别大 | **需要在现场反复调试，大模型帮不上** |
| **姿态估计不准** | 摄像头角度不对、光照变化导致关键点乱跳 | **需要在工厂实际环境调试，大模型帮不上** |
| **性能优化** | 多路视频流处理卡顿、内存泄漏、CPU/GPU 占用过高 | **工程问题，大模型给的是通用方案，不一定能用** |
| **7×24 稳定性** | 跑着跑着就崩了、摄像头断流、进程异常退出 | **运维经验问题，大模型帮不上** |
| **现场部署** | 摄像头安装位置、角度、网络布线、服务器部署 | **纯线下工作，大模型完全帮不上** |

## 三、2 个人会遇到的具体技术坎

### 坎1：环境搭建（预计卡 1~2 周）

```
你需要安装的东西：
├── PaddlePaddle（深度学习框架）
├── PaddleDetection（检测/姿态/跟踪模型）
├── CUDA/cuDNN（如果用GPU）
├── OpenCV + FFmpeg（视频处理）
├── 各种 Python 依赖（版本冲突是常态）
└── 模型权重下载（几个GB，下载+转换）

常见问题：
- PaddlePaddle 版本和 CUDA 版本不匹配 → 装不上
- PaddleDetection 依赖的某个包版本冲突 → 跑不起来
- 模型权重文件格式不对 → 加载失败
- Windows vs Linux 环境差异 → 代码换个系统就跑不了
```

**大模型能帮吗？** 能给你安装命令，但报错信息千奇百怪，大模型经常给出过时的解决方案。

**建议**：直接用 Docker 或者 PaddleDetection 官方提供的预装镜像，能省一半时间。

### 坎2：模型跑通但效果差（预计卡 2~4 周）

```
大模型能帮你写出这样的代码：

    import paddle
    from ppdet.engine import Trainer
    cfg = load_config('configs/ppyoloe/ppyoloe_crn_l_300e_coco.yml')
    trainer = Trainer(cfg, mode='infer')
    trainer.predict('video_frame.jpg')

但你面对的真实问题是：

1. 检测到了人，但 bbox 框不准（框偏了、框太小）
2. 人走到柱子后面再出来，ID 变了（之前是 ID#3，变成 ID#7）
3. 光线一变（开灯/关灯/白天/晚上），检测率骤降
4. 摄像头角度太斜，姿态估计关键点全乱了
5. 人多的时候互相遮挡，漏检严重
```

**大模型能帮吗？** 不能。大模型看不到你的实际摄像头画面，不知道你的工厂环境长什么样。

**你需要做的**：拿着摄像头画面，一个参数一个参数地调，一个场景一个场景地测。

### 坎3：多路视频流处理（预计卡 1~2 周）

```
1 路视频流：简单，OpenCV 直接读
5 路视频流：还行，但 CPU 占用开始飙升
10 路视频流：问题来了
  - 内存不够 → 进程被 OOM Kill
  - 某路摄像头断流 → 整个程序卡死
  - 帧率不一致 → 有的路快有的路慢
  - 线程/进程管理 → 写起来容易出 Bug
```

**大模型能帮吗？** 能给你多线程/多进程的代码框架，但实际跑起来的稳定性问题，只能靠你自己排查。

### 坎4：现场部署（预计 1~2 周）

```
在办公室开发时一切正常，到了工厂现场：
- 网络不通（工厂内网限制）
- 摄像头 IP 段不同
- 服务器没有 GPU（或 GPU 驱动装不上）
- 摄像头角度不是你想要的（装太高/太远/有遮挡）
- 工人不配合（故意挡摄像头、换衣服）
```

**大模型能帮吗？** 完全帮不上。这是纯线下工作。

### 坎5：号码识别（建议放弃）

```
这个功能对你们来说几乎不可能在短期内做好：
1. 需要采集大量衣服号码图片（几百到几千张）
2. 需要人工标注（画框标号码，非常耗时）
3. 需要训练专门的检测模型（训练参数、数据增强、模型选择）
4. 训练出来的模型换个环境就不准了
5. 即使号码检测准了，OCR 识别褶皱/污损的数字也很难

这不是"写代码"的问题，是"数据+调参+经验"的问题。
大模型在这块几乎帮不上忙。
```

## 四、2 个人 + 大模型辅助，到底能做什么？

### 4.1 能做的（建议先做这些）

| 功能 | 技术方案 | 大模型能帮多少 | 预计工期 |
|------|----------|--------------|---------|
| 视频流拉取 + 显示 | OpenCV 读 RTSP → 前端展示 | 90% | 1 周 |
| 人体检测（画框） | YOLO/PP-YOLOE 预训练模型直接推理 | 70% | 1~2 周 |
| 人数统计 + 聚集检测 | 检测人 → 数数 → 超阈值告警 | 80% | 1 周 |
| 区域闯入检测 | ROI 区域 + 人体检测 | 85% | 0.5 周 |
| 人员静止检测（基础版） | 跟踪 + 位移判断 | 60% | 2 周 |
| 后台管理 + 告警 | FastAPI + 前端 + 企微推送 | 95% | 1~2 周 |
| **总计** | | | **5~7 周** |

### 4.2 做不了或做不好的（建议先不做）

| 功能 | 为什么做不了 |
|------|-------------|
| 摔倒检测 | 需要姿态估计调参，现场调试周期长，2 个人忙不过来 |
| 低头检测 | 同上，摄像头角度对姿态估计影响太大 |
| 号码识别 | 需要采集数据、标注、训练模型，不是写代码能解决的 |
| 多路（>5路）稳定运行 | 工程优化经验要求高，容易出稳定性问题 |
| 7×24 长期运行 | 需要运维经验，异常恢复、断流重连等 |

## 五、最务实的执行方案

### 第一步：买一路商业AI摄像头先试（第 1 周）

**花 ¥6,000~20,000 买一路燧机科技的AI摄像头**

- 开箱即用，不需要你调模型
- 自带离岗/睡岗/摔倒/聚集检测
- 直接看实际效果，给副总汇报

**你们要做的事**：安装摄像头 → 接上网线 → 打开管理后台 → 看效果

### 第二步：自研简单功能做 Demo（第 2~6 周）

用你们自己的技术栈 + 大模型辅助，做一个**基础版 Demo**：

```
第 2 周：搭环境 + 跑通 YOLO 人体检测（大模型能帮 70%）
第 3 周：接 RTSP 视频流 + 实时检测（大模型能帮 80%）
第 4 周：聚集检测 + 区域闯入检测（大模型能帮 80%）
第 5 周：FastAPI 后台 + 告警推送（大模型能帮 95%）
第 6 周：前端看板 + 联调（大模型能帮 90%）
```

**Demo 效果**：能检测人员聚集、区域闯入，有告警推送和管理看板

### 第三步：拿着 Demo + 商业摄像头效果给副总汇报（第 6~7 周）

**汇报内容**：

> "我们自己做了一个基础版 Demo，能检测人员聚集和区域闯入。
> 同时买了一路商业摄像头做对比，商业摄像头在摔倒、睡岗检测上效果更好。
>
> 建议：
> - 基础检测（聚集/闯入/人数统计）→ 我们自己做，省成本
> - 复杂检测（摔倒/睡岗/低头）→ 买商业摄像头，效果好
> - 身份识别 → 用 RFID 工牌，不用号码衣服
> - 后台管理 → 我们自己做，可以完全定制化"

## 七、给你们的真心话

1. **别被大模型的能力迷惑**：大模型能帮你写出漂亮的业务代码，但 AI 视觉项目的核心不是"写代码"，是"调模型 + 调参数 + 现场调试"，这些大模型帮不上

2. **2 个人做 Demo 没问题，做生产系统不够**：Demo 只需要"跑通"，生产系统需要"稳定"，后者的工作量是前者的 5~10 倍

3. **号码识别别碰**：对你们团队来说，这是投入产出比最低的功能

4. **最聪明的做法是"买 + 做"结合**：买商业摄像头解决复杂检测，自己做后台和简单检测，既省钱又能出活

5. **先买一路摄像头试效果**：花几千块钱验证一下，比自己闷头开发几个月发现效果不行要强得多
