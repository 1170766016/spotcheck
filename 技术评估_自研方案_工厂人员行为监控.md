### 3.3 行为检测算法设计

#### (1) 长时间站立不动

```
算法逻辑：
1. 人体检测 + ByteTrack 跟踪，获取每个人的 track_id 和 bbox
2. 对每个 track_id，记录 bbox 中心点的位移
3. 如果连续 N 帧（可配置，如 5 分钟 = 7500 帧 @25fps）中心点位移 < 阈值（如 30px）
4. 触发"长时间静止"告警
5. 排除情况：工位本身就需要站立的场景（通过 ROI 区域配置排除）

关键点：跟踪稳定性、遮挡后重新出现的 ID 保持
难度：★★☆☆☆（低）
```

#### (2) 长时间低头

```
算法逻辑：
1. PP-TinyPose 提取人体关键点（头部：鼻子、左眼、右眼、左耳、右耳、颈部）
2. 计算头部俯仰角：通过 鼻子-颈部 的垂直夹角
3. 如果俯仰角 > 阈值（如 45°）且持续 > N 秒（如 30 秒）
4. 触发"长时间低头"告警

关键点：摄像头角度对姿态估计影响大，需针对不同安装角度做校准
难度：★★★☆☆（中）
```

#### (3) 摔倒检测

```
算法逻辑：
1. PP-TinyPose 提取人体关键点
2. 计算人体宽高比（bbox）+ 躯干倾斜角 + 头部高度
3. 规则判定：
   - 人体宽高比突然变化（从竖长变横宽）
   - 头部关键点高度骤降（低于髋部关键点）
   - 躯干与地面夹角 < 30°
4. 连续 3~5 帧确认（避免误报）
5. 触发"摔倒"告警

关键点：蹲下/坐下 vs 摔倒的区分，需要时序分析
难度：★★★☆☆（中）
```

#### (4) 多人聚集

```
算法逻辑：
1. 人体检测，统计 ROI 区域内的人数
2. 如果人数 > 阈值（如 3 人）且持续 > N 秒（如 60 秒）
3. 触发"人员聚集"告警

关键点：最简单的检测项，几乎无技术难点
难度：★☆☆☆☆（极低）
```

#### (5) 衣服号码识别

```
算法逻辑：
1. 人体检测后，裁剪人体区域
2. 训练一个"号码区域检测"模型（YOLO），检测衣服上的号码位置
3. 裁剪号码区域，送入 PaddleOCR 进行号码识别
4. 将号码与人员档案绑定

关键技术难点：
- 号码被遮挡（转身、手臂遮挡）→ 需多帧识别，取众数
- 衣服褶皱导致号码变形 → 数据增强 + 微调
- 远距离号码模糊 → 需要 400 万像素以上摄像头，或限定检测距离
- 号码字体多样 → 需要采集实际衣服号码图片做训练

建议策略：
- 先做"号码检测 + OCR"的 baseline
- 在入口处部署近距离摄像头专门采集号码（上岗时刷号绑定）
- 工作区域内通过 ReID（外观特征重识别）做跨帧追踪，减少号码依赖

难度：★★★★☆（较高，是本项目最大技术挑战）
```

## 八、号码衣服方案的技术问题深度分析

### 8.1 核心矛盾：号码识别比人脸识别更难，不是更简单

老板的逻辑是：**号码衣服 → 不用人脸识别 → 省成本**。这个逻辑表面上合理，但技术上有个根本性误解。

| 对比 | 人脸识别 | 衣服号码识别 |
|------|----------|-------------|
| 目标特征 | 脸部（固定位置，特征丰富，每人唯一） | 衣服上的数字（位置不固定，字体多样） |
| 可见性 | 正面/侧面都可见 | 转身就看不到，手臂一抬就挡住 |
| 受环境影响 | 光照 | 光照 + 褶皱 + 污损 + 遮挡 |
| 所需摄像头 | 200 万像素即可 | **400 万像素以上（要看清数字笔画）** |
| 成熟度 | 极其成熟（调包即用，准确率 99%+） | **无成熟方案（需自研）** |
| 成本 | 低 | **反而更高** |

### 8.2 具体技术问题清单

| 问题 | 说明 | 严重程度 |
|------|------|----------|
| **转身不可见** | 号码在胸前或背后，工人一转身摄像头就拍不到，一个人有约 50% 的时间号码不可见 | 严重 |
| **遮挡** | 手臂抬起、弯腰、拿工具时都会挡住号码 | 严重 |
| **衣服褶皱变形** | 号码印在布料上，一活动就皱，数字 6 和 8、1 和 7 容易混淆 | 中等 |
| **油污灰尘** | 工厂环境污损快，号码用几天就看不清了 | 中等 |
| **距离问题** | 5 米外要看清衣服上 5cm 的数字，需要 400 万像素以上，**省不了摄像头钱** | 中等 |
| **多人重叠** | 聚集场景下人挤人号码互相挡住，恰恰是最需要识别身份的时候 | 严重 |
| **换衣服/穿错** | 衣服脏了换一件、两个人穿错，号码绑定就乱了 | 中等 |

> **一句话总结：号码识别不是"简化版的人脸识别"，而是一个比人脸识别更难的计算机视觉问题。以为能省成本，实际上反而要花更多钱在摄像头和算法开发上。**

### 8.3 号码衣服的正确用法

号码衣服**不是不能用**，但不要依赖摄像头去识别号码，换一种用法：

> 号码衣服 = **视觉辅助标识**，方便管理人员**肉眼看录像回放时快速定位是谁**

实际的身份绑定靠 **RFID 刷卡** 完成。这样号码衣服的价值还在，但系统不用去解决"识别衣服号码"这个技术难题。

## 九、身份绑定的替代方案

老板的**真实需求**是：**低成本锁定具体是哪个人，不用人脸**。满足这个需求的方案有好几个：

### 9.1 方案对比

| 方案 | 原理 | 成本 | 可靠性 | 老板能接受吗 |
|------|------|------|--------|------------|
| **RFID 工牌** | 每人挂一个工牌（工卡大小），内有芯片，经过读卡器自动识别 | 50~200元/个 | 极高 | 能接受（和号码衣服思路一样，不采集人脸） |
| **颜色工位帽 + 编号** | 不同颜色代表不同班组，帽子上贴大编号 | 10~30元/顶 | 中高 | 能接受 |
| **上岗扫码绑定** | 工人上岗时在工位扫码登记，系统记录"这个工位 = 张三" | 0元 | 高 | 能接受（最简单） |
| **号码衣服（原方案）** | 衣服印号码，摄像头识别 | 衣服20~50元/件 + 算法开发 | 低 | 技术风险高 |

### 9.2 最推荐：RFID 工牌方案

这是最符合老板诉求的方案：

1. **不采集人脸** — 和号码衣服一样满足"不要人脸"的要求
2. **极其成熟** — 门禁系统、公交卡、食堂刷卡都是这个技术，零技术风险
3. **成本低** — 工牌 50~200 元/个，100 人也就 0.5~2 万，比开发号码识别算法便宜几十倍
4. **使用简单** — 工人上岗时刷一下卡，系统自动绑定"工牌号 = 人员 = 所在区域"
5. **不受遮挡影响** — RFID 是射频信号，不存在"转身看不到"的问题

**具体使用流程：**
```
工人上岗 → 在工位/入口处刷 RFID 工牌 → 系统记录：工牌007 = 张三，在A区域
摄像头检测到 A 区域有人行为异常 → 系统查绑定记录 → 告警：张三长时间静止
工人换岗/下班 → 再刷一次卡 → 系统更新绑定关系
```

### 9.3 向副总汇报的话术建议

> "号码衣服的方案我们评估过了，技术上有个问题：**摄像头要看清衣服上的数字，比看清人脸还难**，因为人会转身、号码会被手臂挡住、衣服会皱。如果用这个方案，摄像头反而要买更高清的，算法也要专门定制开发，成本比预想的高。
>
> **我们建议用 RFID 工牌替代**，每个工人挂一个工卡大小的牌子，上岗时刷一下，系统就自动绑定了。这个技术和公交卡、门禁卡一样成熟，一个工牌几十块钱，100 个人也就几千块，不需要高清摄像头，也不用采集人脸。
>
> 号码衣服也可以同时穿，方便管理人员看录像回放时快速辨认是谁，**两套方案互补，不冲突**。"

## 十、结论与建议

### 10.1 我们能自己做吗？

**结论：完全可以。**

- 团队现有技术栈（PaddlePaddle、OpenCV、FastAPI）与本项目高度匹配
- PaddleDetection 提供了开箱即用的人体检测、姿态估计、目标跟踪模型
- 行为分析（静止/低头/摔倒/聚集）第一阶段可用规则引擎实现，技术门槛不高
- 号码识别是最大挑战，但团队有 OCR 经验，可在此基础上适配
- 全开源技术栈，软件成本为零

### 10.2 推荐实施路径

1. **立刻启动第一阶段（MVP）**：2 人 × 6~8 周，先把行为检测跑通
2. **MVP 先不做号码识别**：用 track_id 标识人员，验证行为检测效果
3. **向副总汇报时带上 RFID 备选方案**：号码衣服 + RFID 双保险
4. **第一阶段完成后现场试用**：收集真实场景数据，指导第二阶段开发
5. **第二阶段再做号码识别**：有了现场数据，针对性训练模型

### 10.3 给副总的汇报要点

- **自研 vs 外采可节省 ¥10~30 万**，且完全自主可控
- **3~4 个月出完整系统**，第一阶段 2 个月内可见初步效果
- **硬件投入仅 ¥3~7 万**（10 路摄像头规模），软件零成本
- 系统可作为部门技术能力展示，也可复用到其他工厂/车间
